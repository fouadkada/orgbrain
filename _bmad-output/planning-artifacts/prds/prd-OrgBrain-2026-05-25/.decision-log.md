### Decisions made in coaching session (2026-05-25)

| Decision | Choice | Rationale |
|---|---|---|
| V1 integration stack | Slack (public channels) + Google Workspace (Meet, Drive) | Microsoft 365 is V2 (maybe) |
| Slack privacy boundary | Public channels only — hard limit, not configurable | DMs create legal and trust exposure; clear boundary easier to communicate to employees |
| Historical import model | Admin-triggered with review-before-confirm | Gives org control; review step is the consent moment for historical data |

| Confidence below threshold | Decline and route to Knowledge Owner — never surface low-confidence answer | False confidence is worse than "I don't know" |
| Departure risk signals | 4 clusters: engagement velocity, meeting participation, calendar activity, future-orientation | Composite trend (3+ clusters, 30-day window) triggers signal |
| Engineering metrics (PRs, LOC) | V2/V3 via GitHub/GitLab | Out of V1 data source scope |
| Pricing tiers | Base + Intelligence; manager tier dropped for V1 | Simpler launch, cleaner value prop |
| Pricing model | Land-and-expand; per-org flat pricing assumed | Specific price TBD (Open Question 1) |
| Pinning permission | Any Member can pin — no role or approval required | Democratized curation |
| Intelligence Dashboard surface | Web UI only in V1; Slack digest TBD | VP persona is less Slack-native than ICs |
| Buyer persona | VP of Engineering (primary), CEO at smaller orgs | People/HR team is influencer, not budget holder |
